# CellTech Backend Setup Guide

## Project Structure

```
celltech-backend/
├── prisma/
│   ├── schema.prisma          # Your Neon PostgreSQL schema
│   ├── seed.ts                # Seed data (brands, models, inventory)
│   └── migrations/            # Auto-generated by Prisma
├── src/
│   ├── server.ts              # Express server entry point
│   ├── routes/
│   │   ├── parts.ts           # GET /api/parts (search)
│   │   ├── inventory.ts       # GET /api/inventory/:skuId
│   │   ├── compatibility.ts   # GET /api/compatibility/:skuId
│   │   ├── brands.ts          # GET /api/brands
│   │   └── models.ts          # GET /api/models
│   ├── middleware/
│   │   ├── errorHandler.ts
│   │   └── corsSetup.ts
│   └── utils/
│       └── validators.ts
├── public/
│   ├── index.html             # Test dashboard
│   ├── styles.css             # Dashboard styling
│   └── app.js                 # Dashboard interactivity
├── .env.example
├── .env                       # Your local env (DON'T COMMIT)
├── package.json
├── tsconfig.json
└── README.md
```

## Setup Steps

### Step 1: Initialize Project

```bash
mkdir celltech-backend
cd celltech-backend
npm init -y
```

### Step 2: Install Dependencies

```bash
npm install @prisma/client prisma express cors dotenv
npm install -D typescript ts-node @types/express @types/node @types/node
npm install -D nodemon tsx
```

### Step 3: Create tsconfig.json

```bash
npx tsc --init
```

Modify to:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "lib": ["ES2020"],
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true
  }
}
```

### Step 4: Set Up Prisma

```bash
npx prisma init
```

This creates `prisma/.env`. Edit it:

```
DATABASE_URL="postgresql://user:password@host:5432/celltech?schema=public"
```

Get your Neon connection string from dashboard:

- Go to neon.tech
- Select your project
- Copy the connection string (looks like: `postgresql://...`)

### Step 5: Create schema.prisma

See **schema.prisma** file below.

### Step 6: Push Schema to Neon

```bash
npx prisma migrate dev --name init
```

This creates the tables in Neon.

### Step 7: Seed Your Data

```bash
npx ts-node prisma/seed.ts
```

Now your Neon database is populated with brands, models, inventory, etc.

### Step 8: Run Locally

```bash
npm run dev
```

Visit: `http://localhost:3000`

You’ll see the test dashboard.

## Deployment

### Railway Deployment

1. Push code to GitHub
1. Go to railway.app
1. Select “New Project” → “Deploy from GitHub”
1. Connect repo
1. Add environment variables:
- `DATABASE_URL` (from Neon)
- `NODE_ENV=production`
- `CORS_ORIGIN=https://your-frontend.vercel.app`
1. Railway auto-deploys on every push

Your backend will be at: `https://your-railway-app.up.railway.app`

### Next.js Frontend Integration

In your Next.js repo, create `lib/api.ts`:

```typescript
const API_URL = process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3000';

export async function searchParts(device: string) {
  const res = await fetch(`${API_URL}/api/parts?device=${encodeURIComponent(device)}`);
  if (!res.ok) throw new Error('Failed to search parts');
  return res.json();
}

export async function getPartDetails(skuId: string) {
  const res = await fetch(`${API_URL}/api/inventory/${skuId}`);
  if (!res.ok) throw new Error('Failed to fetch part details');
  return res.json();
}

export async function getCompatibility(skuId: string) {
  const res = await fetch(`${API_URL}/api/compatibility/${skuId}`);
  if (!res.ok) throw new Error('Failed to fetch compatibility');
  return res.json();
}

export async function getBrands() {
  const res = await fetch(`${API_URL}/api/brands`);
  if (!res.ok) throw new Error('Failed to fetch brands');
  return res.json();
}

export async function getModels(brandId?: number) {
  const url = brandId 
    ? `${API_URL}/api/models?brandId=${brandId}`
    : `${API_URL}/api/models`;
  const res = await fetch(url);
  if (!res.ok) throw new Error('Failed to fetch models');
  return res.json();
}
```

In `.env.local`:

```
NEXT_PUBLIC_API_URL=https://your-railway-backend.up.railway.app
```
